Reinforcement Learning with Verifiable Rewards (RLVR) for Qwen VLM – Image Modality

In this lab you will apply RLVR (Reinforcement Learning with Verifiable Rewards) to a Qwen2-VL vision-language model so it learns to reason over images using rule-based reward signals— no human preference labels required. You will fine-tune the model on a real-world image Q&A dataset (ChartQA), evaluate multimodal reasoning quality, deploy the model to a SageMaker endpoint, and then clean up all provisioned resources.

Lab progress0 / 5 steps

What is RLVR for Vision-Language Models?

RLVR extends the ideas behind DeepSeek-R1 and Qwen's RLVR text work to the multimodal domain. Instead of scoring model outputs with a trained reward model, RLVR uses deterministic, verifiable reward functions that compare the model's answer against a ground-truth label extracted directly from the dataset. For image tasks this means:

  • The model receives an image together with a natural-language question.
  • It must emit structured reasoning (<think>…</think>) followed by a short answer.
  • A reward function checks whether the answer matches the verified ground truth (exact match, numeric tolerance, or format compliance).
  • The GRPO algorithm uses these scalar rewards to update the vision-language model's weights.

1.4  Convert to Qwen2-VL Conversation Format

Qwen2-VL's processor expects messages in a structured multi-turn format that interleaves image and text content blocks. We also inject the RLVR system prompt that instructs the model to wrap its reasoning in <think> tags.

Python – cell 3: format_sample helper notebook 
SYSTEM_PROMPT = """You are a visual reasoning assistant.
When given an image and a question, first reason carefully inside
<think>...</think> tags, then give ONLY the final short answer.

Format:
<think>
[Your step-by-step reasoning here]
</think>
[Final short answer – no more than a few words or a number]"""

def format_sample(example):
    """Convert a ChartQA example to Qwen2-VL conversation format."""
    return {
        "messages": [
            {
                "role": "system",
                "content": SYSTEM_PROMPT,
            },
            {
                "role": "user",
                "content": [
                    {"type": "image"},           # image placeholder
                    {"type": "text",
                     "text": example["query"]},
                ],
            },
        ],
        "image":    example["image"],        # PIL Image object
        "answer":   example["label"][0],     # ground-truth string
    }

# Apply formatting
train_ds = raw["train"].map(format_sample, remove_columns=raw["train"].column_names)
val_ds   = raw["val"].map(format_sample,   remove_columns=raw["val"].column_names)
print(f"Train: {len(train_ds)} | Val: {len(val_ds)}")

Fine-Tuning with RLVR (GRPO)

This step fine-tunes Qwen2-VL-7B-Instruct using the Group Relative Policy Optimization (GRPO) algorithm from the TRL library. GRPO samples multiple candidate responses per prompt, scores them with verifiable reward functions, and uses the relative rewards to update the model policy—no separate critic network required.

⚠️
This step requires a GPU instance. In SageMaker Studio, switch your notebook kernel to a ml.g5.12xlarge instance before running the cells below. Estimated training time: ~90 minutes for 3 epochs on 28K chart images.

2.1  RLVR Reward Functions

The core of RLVR is a set of deterministic reward functions. For chart Q&A we define three complementary rewards that are summed to produce a scalar signal per rollout.

Python – cell 6: reward functions notebook 
import re, math

def extract_answer(text: str) -> str:
    """Strip <think> block; return everything after it."""
    text = re.sub(r"<think>.*?</think>", "", text, flags=re.DOTALL)
    return text.strip()

def reward_exact_match(completions, ground_truth, **kwargs):
    """+1 if answer exactly matches ground truth (case-insensitive)."""
    rewards = []
    for c in completions:
        pred = extract_answer(c[0]["content"]).lower().strip()
        gt   = ground_truth.lower().strip()
        rewards.append(1.0 if pred == gt else 0.0)
    return rewards

def reward_numeric_tolerance(completions, ground_truth, tol=0.05, **kwargs):
    """Partial credit when numeric answer is within ±5% of ground truth."""
    rewards = []
    for c in completions:
        pred_txt = extract_answer(c[0]["content"])
        try:
            pred = float(re.search(rr"[-+]?\d*\.?\d+", pred_txt).group())
            gt   = float(re.search(rr"[-+]?\d*\.?\d+", ground_truth).group())
            rewards.append(1.0 if abs(pred - gt) / (abs(gt) + 1e-9) <= tol else 0.0)
        except:
            rewards.append(0.0)
    return rewards

def reward_format(completions, **kwargs):
    """Small reward for correct <think>…</think> + answer format."""
    rewards = []
    pattern = re.compile(r"<think>.+?</think>\s*\S", re.DOTALL)
    for c in completions:
        text = c[0]["content"]
        rewards.append(0.1 if pattern.search(text) else 0.0)
    return rewards

print("✅ Reward functions defined")

2.4  Combined Reward Wrapper

Python – cell 9: combined reward notebook 
def combined_reward(completions, ground_truth, **kwargs):
    """
    Aggregate reward = exact_match  (weight 1.0)
                     + numeric_tol  (weight 0.5, additive if exact fails)
                     + format       (weight 0.1)
    Clipped to [0, 1].
    """
    em   = reward_exact_match(completions, ground_truth)
    num  = reward_numeric_tolerance(completions, ground_truth)
    fmt  = reward_format(completions)

    combined = []
    for i in range(len(completions)):
        # Numeric tolerance only contributes when exact match fails
        r = em[i] + (0.5 * num[i] if em[i] == 0 else 0) + fmt[i]
        combined.append(min(r, 1.0))
    return combined

3.3  Run Evaluation Loop

Python – cell 13: evaluation notebook 
from tqdm import tqdm

test_ds = raw["test"].map(format_sample, remove_columns=raw["test"].column_names)

exact_hits, relaxed_hits, fmt_hits = 0, 0, 0
results = []

merged.eval()
with torch.inference_mode():
    for i, sample in enumerate(tqdm(test_ds)):
        text   = processor.apply_chat_template(
                     sample["messages"], tokenize=False,
                     add_generation_prompt=True)
        imgs, _ = process_vision_info(sample["messages"])
        inputs  = processor(text=[text], images=imgs,
                            return_tensors="pt").to("cuda")

        out_ids = merged.generate(**inputs, max_new_tokens=512,
                                   temperature=0.01, do_sample=False)
        response = processor.decode(out_ids[0][inputs.input_ids.shape[1]:],
                                    skip_special_tokens=True)
        pred  = extract_answer(response).lower().strip()
        gt    = sample["answer"].lower().strip()

        em  = int(pred == gt)
        ra  = int(reward_numeric_tolerance([[{"content": response}]], gt)[0] > 0)
        fmt = int(bool(re.search(r"<think>.+?</think>", response, re.DOTALL)))

        exact_hits   += em
        relaxed_hits += ra or em
        fmt_hits     += fmt
        results.append({"pred": pred, "gt": gt, "em": em, "ra": ra})

N = len(test_ds)
print(f"\n{'Metric':<25}{'Score':>10}")
print("-" * 36)
print(f"{'Exact Match':<25}{exact_hits/N*100:>9.1f}%")
print(f"{'Relaxed Accuracy (+5%)':<25}{relaxed_hits/N*100:>9.1f}%")
print(f"{'Format Compliance':<25}{fmt_hits/N*100:>9.1f}%")
