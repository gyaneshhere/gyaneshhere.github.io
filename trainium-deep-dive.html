Core Architecture

Built for Training at Scale

Trainium integrates 16 NeuronCore v2 processors per chip, each with dedicated compute engines and a high-bandwidth scratchpad, all connected via a custom intra-chip fabric.

⬡

NeuronCore v2

Each chip integrates 16 NeuronCore v2 units. Every core contains a high-throughput Tensor engine (matrix multiply), a Vector engine (element-wise ops), a Scalar engine (control logic), and a dedicated SRAM scratchpad for near-zero latency data access.

Tensor EngineVector EngineScalar EngineSRAM Scratchpad
▪

HBM Memory

High Bandwidth Memory is stacked directly on the Trainium die. Each chip provides 32 GB of HBM across four stacks. The trn1.32xlarge pairs 16 chips for 512 GB total — enabling very large model training without offloading to host memory.

32 GB per chip512 GB on trn1.32xlargeHigh bandwidth
⇄

NeuronLink

NeuronLink is AWS's custom intra-node interconnect between Trainium chips. It delivers significantly higher bandwidth than PCIe for AllReduce operations, enabling efficient data-parallel and model-parallel training within a single instance.

Intra-nodeAllReduceLow latency
⌬

Elastic Fabric Adapter

The trn1.32xlarge provides 8 × 100 Gbps EFA interfaces (800 Gbps total) for inter-node communication. trn1n.32xlarge doubles this to 1,600 Gbps. EFA supports NCCL and MPI collectives for distributed training across hundreds of nodes.

Distributed Training

Parallelism Strategies

NxD Training and NxD Core provide first-class support for all parallelism strategies, enabling efficient training of models with trillions of parameters.

Data Parallelism

Each NeuronCore processes a separate data batch. The full model is replicated across all workers. Gradients are synchronized via AllReduce after each backward pass — over NeuronLink intra-node or EFA inter-node.

Supported natively via torch.nn.parallel.DistributedDataParallel. Scales linearly with chip count.

AllReduceZeRO-1NeuronLinkLinear scaling

Tensor Parallelism

Large weight matrices are sharded across multiple NeuronCores. Each core computes its partition of the matrix multiply; results are assembled with AllGather.

Provided by NxD Core's ColumnParallelLinear / RowParallelLinear APIs. Ideal for attention heads in transformer models.

Weight shardingAllGatherColumnParallelRowParallel

Pipeline Parallelism

Model layers are partitioned into stages across devices. Uses 1F1B scheduling (one-forward-one-backward) with micro-batches to keep all stages busy.

NxD Training supports automatic stage partitioning. Activation recomputation reduces HBM usage by ~60%, enabling much larger models per chip.

1F1B scheduleMicro-batchingActivation recomputeAuto-partitioning

ZeRO Optimizer

Zero Redundancy Optimizer shards optimizer states across workers, achieving near-linear memory reduction with world size. ZeRO-1 is natively supported in NxD Training.

Enables 70B+ parameter model training on a single trn1.32xlarge by dramatically reducing per-rank optimizer state memory from N to N/world_size.
