RadixAttention & Prefix Caching

Traditional engines discard KV caches after each request — even when prompts share identical prefixes. SGLang's radix tree retains and reuses them. Every cache hit saves prefill computation proportional to the shared prefix length.

# SGLang RadixAttention — auto KV reuse import sglang as sgl # All 3 requests share the same system prompt. # SGLang prefills it ONCE, reuses on req 2 & 3. @sgl.function def ask(s, question): s += sgl.system("You are a helpful assistant.") s += sgl.user(question) # only new tokens s += sgl.assistant(sgl.gen("answer")) # Batch: prefix matched, cache reused ✓ states = ask.run_batch([ {"question": "What is ML?"}, {"question": "Explain backprop."}, {"question": "What is a GPU?"}, ])

The radix tree stores KV tensors at every node. When a new request arrives, SGLang performs a longest-prefix match. If found, only the new suffix tokens need prefilling — skipping potentially thousands of GPU operations.

Memory is managed with an LRU eviction policy — leaves are evicted first, preserving shared ancestor nodes as long as possible. This is the key insight: ancestors are worth more because they're shared by many requests.

Speculative Decoding

A small draft model generates multiple candidate tokens per step. The large target model verifies them in parallel — accepting correct drafts and rejecting the first wrong one. Each accepted draft token is effectively "free".

# Enable speculative decoding in SGLang python -m sglang.launch_server \ --model meta-llama/Llama-3.1-70B \ --speculative-algo EAGLE \ --speculative-draft lmzheng/sglang-EAGLE-LLaMA3.1-70B \ --speculative-num-draft-tokens 4 \ --port 8000 # EAGLE-2 adaptive speculative decoding python -m sglang.launch_server \ --model meta-llama/Llama-3.1-8B \ --speculative-algo EAGLE2 \ --speculative-num-steps 3

SGLang supports EAGLE and EAGLE-2 speculative decoding, plus a Medusa-style variant. The draft model runs in the same CUDA stream and batch as the main model, minimizing latency overhead.

Adaptive speculative decoding (EAGLE-2) dynamically adjusts the number of draft tokens based on observed acceptance rates — maximizing throughput without sacrificing quality on harder tokens.

The Full Optimization Stack

SGLang combines every major LLM inference technique into a single coherent runtime.

🌳
RadixAttention
Radix-tree KV cache with LRU eviction for automatic cross-request prefix sharing. The founding innovation of SGLang.
⚡
Zero-Overhead Scheduler
CPU scheduler runs in a tight loop with ~0ms overhead. Cache-aware priority queuing ensures maximum cache hit rates.
🔍
Speculative Decoding
EAGLE and EAGLE-2 draft models speculatively generate tokens, verified in parallel by the target model.
📄
Structured Outputs
Compressed finite state machines constrain generation to JSON, regex, or custom grammars with near-zero overhead.
✂️
Chunked Prefill
Long prompts are split into chunks that interleave with decode steps — eliminating prefill-decode interference.
🔀
PD Disaggregation
Prefill and decode phases run on separate GPU pools for optimal resource utilization in large deployments.
🧮
Quantization
FP4, FP8, INT4, AWQ, and GPTQ quantization. Quantized KV cache cuts memory bandwidth by up to 4×.
🔗
Multi-LoRA Batching
Serve hundreds of LoRA adapters simultaneously in a single batch without adapter switching overhead.
📡
OpenAI-Compatible API
Drop-in replacement for OpenAI API. Works with any existing OpenAI SDK client without code changes.
