Compute Engines

NeuronCore-v3 Architecture

Four specialized compute engines operate in parallel within each NeuronCore-v3. The Tensor Engine alone achieves 158 FP8 TFLOPS of dense compute, with native FP8 double-throughput mode via a 256×128 systolic array.

Tensor Engine
158 FP8 TFLOPS
Systolic array for GEMM, CONV, and Transpose. A 256×128 FP8 contraction dimension delivers double the throughput of BF16/FP16. Native FP8_E4 and FP8_E5 support. FP8 sparse mode delivers 316 TFLOPS. New explicit transpose mode correctly handles NaN/Inf values.
256×128 FP8 Array 158 dense TFLOPS 316 sparse TFLOPS
Vector Engine
High-BW SIMD
Processes element-wise operations: activation functions, normalization, reduction, and transcendentals. Updated datapath width and frequency improve throughput over v2. Processes output from the Tensor Engine PSUM buffer directly.
Element-wise ops Activations Normalization
Scalar Engine
Control Logic
Handles scalar (1-to-1) operations including address computation, loop control, and branching. Acts as the orchestration layer that schedules work across the other three engines and coordinates HBM/SBUF data movement.
Scheduling Branching DMA control
GpsimdE — New
307 GB/s DMA
General-Purpose SIMD Engine with integrated DMA capabilities. Each GpsimdE processor can read from or write to any SBUF/HBM on-chip or off-chip within the same Trn2 instance. Total integrated DMA bandwidth: 307 GB/s (153 GB/s per direction).
GPSIMD Integrated DMA 307 GB/s total
SBUF upgrade: NeuronCore-v3 increases on-chip SRAM scratchpad (SBUF) to 28 MiB (128 partitions × 224 KiB), up from 24 MiB in v2. With LNC-2 (paired cores), the effective SBUF doubles to 56 MiB per logical core — critical for fitting large attention head tensors in-cache.

Distributed Training

Parallelism Strategies

Data Parallelism

Each NeuronCore (or group) processes a separate data batch. The model is replicated across workers; gradients are synchronized via AllReduce over NeuronLink (intra-node) or EFAv3 (inter-node). ZeRO-1 shards optimizer states to reduce memory pressure.

Supported natively via DistributedDataParallel. Scales linearly with chip count through UltraCluster.

AllReduceZeRO-1NeuronLinkEFAv3

Tensor Parallelism

Weight matrices are sharded across NeuronCores. Each core computes its partition; results are gathered with AllGather. Enabled by NxD Core's ColumnParallelLinear and RowParallelLinear.

Especially effective on Trainium2 with LNC-2: pairs of cores share SBUF for larger effective working set per partition, reducing HBM pressure for very large transformer layers.

Weight shardingAllGatherLNC-2 amplified

Pipeline Parallelism

Model layers are partitioned into stages across chips or instances. Uses 1F1B scheduling with micro-batches to maintain high utilization across all pipeline stages simultaneously.

NxD Training provides automatic stage partitioning and activation recomputation — reducing HBM requirements by up to 60%, enabling models beyond Trn2's physical HBM capacity.

1F1B scheduleMicro-batchingActivation recompute-60% HBM

Context Parallelism

Sequence length is sharded across NeuronCores, enabling training with extremely long context windows (128K+ tokens) that wouldn't fit in a single core's memory.

A new addition in NxD Core v2, Context Parallelism is critical for long-context reasoning models and multi-modal architectures with large visual token sequences. Combines well with TP and PP.
