Overview

What is AWS Neuron?

AWS Neuron is a software development kit (SDK) enabling high-performance deep learning and generative AI on AWS Inferentia and Trainium — AWS's purpose-built ML accelerator chips.

It covers the complete ML development lifecycle: build, train, optimize, and deploy. Whether you're a researcher experimenting with novel architectures, an ML engineer scaling production inference, or a performance engineer squeezing every FLOP, Neuron has a path for you.

Built on an open-source foundation under Apache 2.0, Neuron integrates natively with PyTorch, JAX, and TensorFlow. Frameworks like Hugging Face Transformers, vLLM, PyTorch Lightning, and TorchTitan work directly on Trainium without modification.

neuronx-cc
Graph Compiler

Converts XLA and MLIR computation graphs into NEFF (Neuron Executable File Format) binaries optimized for Trainium/Inferentia. Handles operator fusion, tiling, memory layout, and mixed-precision lowering automatically.

Neuron Runtime
Hardware Abstraction

The NRT layer executes NEFF executables on Trainium and Inferentia chips, managing memory allocation, DMA transfers, and kernel dispatch. Exposed through the Neuron Driver kernel module for EC2 integration.

Neuron Kernel Interface (NKI)

For performance engineers who need full hardware control. NKI exposes the complete Trainium ISA with APIs for memory, scheduling, and instruction-level kernel development.

Direct access to Trainium instruction set architecture. Control instruction-level programming, memory allocation, and execution scheduling without abstraction overhead.

Built on MLIR, the open-source NKI Compiler gives full visibility into the complete compilation pipeline from high-level operations to hardware instructions. Apache 2.0 licensed.

NKI Library (NKILib)
Open-source library of pre-optimized kernel implementations for attention, MLP, and normalization operations. Ships with source code, documentation, and benchmarks.

Structured API namespace with updated language constructs introduced in Neuron 2.27. Cleaner ergonomics for custom kernel authoring, with direct hardware intrinsics available.

# Example NKI custom kernel (nki.* namespace)
import neuronxcc.nki as nki
import neuronxcc.nki.language as nl
import neuronxcc.nki.isa as nisa

@nki.jit
def fused_rmsnorm(x, weight, eps=1e-6):
    # Allocate SBUF (on-chip SRAM) tiles
    x_tile  = nl.load(x)
    w_tile  = nl.load(weight)

    # Compute RMS norm on Tensor Engine
    sq_mean = nl.mean(
        nisa.tensor_scalar(x_tile, x_tile, "mul"),
        axis=[-1]
    )
    rstd    = nisa.rsqrt(sq_mean + eps)
    normed  = nisa.tensor_scalar(x_tile, rstd, "mul")
    out     = nisa.tensor_tensor(normed, w_tile, "mul")

    # Store result from SBUF to HBM
    nl.store(out_ref, out)
    return out_ref
# Native PyTorch training on Trainium (TorchNeuron)
import torch
import torch_neuronx

# Trainium registers as a native device
device = torch.device("xla")  # → neuron device

# FSDP distributed training — unchanged from GPU
from torch.distributed.fsdp import FullyShardedDataParallel
model = FullyShardedDataParallel(
    MyLlamaModel().to(device)
)

# torch.compile works directly
model = torch.compile(model, backend="neuron")

for batch in dataloader:
    tokens = batch["input_ids"].to(device)
    loss   = model(tokens).loss
    loss.backward()
    optimizer.step()
# Serve LLMs with vLLM on Trainium — zero changes
# $ pip install vllm-neuronx

from vllm import LLM, SamplingParams

# Standard vLLM V1 API — runs on Trainium/Inferentia
llm = LLM(
    model="meta-llama/Llama-3.1-70B-Instruct",
    device="neuron",
    tensor_parallel_size=32,  # NeuronCores
    max_model_len=8192,
)

params = SamplingParams(
    temperature=0.7, top_p=0.95, max_tokens=512
)

# Expert Parallelism for MoE models (e.g. Llama 4)
llm_moe = LLM(
    model="meta-llama/Llama-4-Scout-17B",
    device="neuron",
    expert_parallel_size=4,
)

Framework & Library Integrations

Neuron integrates natively across the modern ML ecosystem — use what you know, on AWS silicon.

🔥
PyTorch
● Native support · TorchNeuron
TorchNeuron registers Trainium as a native PyTorch device. Eager mode, torch.compile, FSDP, DDP, DTensor, and TorchTitan all work without code changes. Migrating to PyTorch 2.10+ native backend (XLA phase-out).
🔷
JAX
● jax-neuronx
JAX developers use jax-neuronx to develop, optimize, and deploy models to both Inferentia and Trainium with familiar JAX workflows. Supports JIT compilation and distributed JAX patterns.
🤗
Hugging Face
● Optimum Neuron
Optimum Neuron bridges Hugging Face Transformers and the Neuron SDK, providing standard HF APIs on Trainium and Inferentia. Supports training, fine-tuning, and inference pipelines.
⚡
vLLM
● vLLM V1 API
Full vLLM V1 API compatibility on Trainium and Inferentia. Expert Parallelism for MoE, disaggregated inference, speculative decoding, continuous batching, and PagedAttention out of the box.
⛓️
PyTorch Lightning
● Integrated
PyTorch Lightning callbacks and trainers work with Trainium for distributed training. Multi-node setups with Lightning + NxD Training provide clean abstractions without sacrificing hardware performance.
🔗
TorchTitan
● Zero modifications
TorchTitan, Meta's LLM training framework, runs directly on Trainium via TorchNeuron — no fork, no patch. Production-ready Llama pre-training workflows translate without changes.
🌐
TensorFlow
● tensorflow-neuron
tensorflow-neuron provides Inferentia support for TensorFlow models. Compile and serve TF SavedModels with Neuron runtime for low-latency production inference workloads.
🎛️
Anyscale Ray
● Third-party
Ray clusters on Trainium instances enable distributed training and inference orchestration at scale, with Neuron device management handled transparently by the Neuron device plugin.

Release Notes

What's New

Latest changes across the Neuron SDK — from Trainium3 support to the new open-source NKI compiler and native PyTorch backend.

Neuron 2.28.1
Released March 13, 2026
Transition to native PyTorch support (TorchNeuron). PyTorch 2.9 is the last release using PyTorch/XLA — upgrading to PyTorch 2.10+ will use the new native backend. End-of-support notices for NxDT/NxD Core training APIs starting PyTorch 2.10.
NEW
Trainium3 UltraServer support — Full Trn3 instance and UltraServer support with NeuronSwitch-v1 for all-to-all collectives across 144 chips (Neuron 2.27.0+).
NEW
TorchNeuron (Native PyTorch backend) — Registers Trainium as a PrivateUse1 native device. Run PyTorch code unchanged. Open source, Apache 2.0 (private beta → GA in 2.29).
NEW
NKI Compiler open-sourced (MLIR) — Full compiler pipeline visibility from high-level ops to hardware instructions. New nki.* namespace with updated APIs and language constructs.
NEW
NKI Library — Open-source library of production-optimized kernels for attention, MLP, and normalization with benchmarks and source code.
NEW
Neuron Explorer v2 — Four-viewer profiling suite (Hierarchy, AI Recommendations, Source Code, Summary). Available via UI, CLI, and VSCode IDE.
BETA
Neuron DRA (Dynamic Resource Allocation) — Kubernetes-native resource management for NeuronCores. Replaces static device plugin allocation for fine-grained workload scheduling.
BETA
Expert Parallelism for MoE models — Efficient distribution of MoE experts across NeuronCores. Supports Llama 4 Scout and Maverick on Trn2 instances.
NEW
PyTorch 2.9 / JAX 0.6.2 support — Latest framework support with improved inference capabilities on Trn2 instances. FLUX.1-dev image generation supported on Trn2.
