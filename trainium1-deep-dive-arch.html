Overview

What is AWS Trainium?

AWS Trainium is the second-generation purpose-built ML accelerator from AWS, designed from the ground up for deep learning training of large-scale generative AI models including LLMs, diffusion models, and recommendation systems.

🎯

Purpose-Built for Training

Unlike general-purpose GPUs, Trainium's NeuronCore-v2 architecture is specifically optimized for large-scale DL training with dedicated collective communication hardware, stochastic rounding, and configurable FP8 data types.

cFP8BF16TF32

Up to 50% Cost Savings

Trn1 instances offer up to 50% cost-to-train savings over comparable EC2 GPU instances, enabling organizations to train 100B+ parameter models at dramatically reduced cost.

Configurable FP8 (cFP8): Trainium introduced cFP8 — a novel data type where the exponent/mantissa split is configurable per-layer by the compiler, enabling higher accuracy than fixed FP8 while retaining the throughput benefits of 8-bit computation. This is particularly effective for transformer attention layers which have differing numerical ranges across layers.

Compute Engine

NeuronCore-v2 Deep Dive

Each Trainium chip contains two second-generation NeuronCore-v2 engines. Unlike GPU architectures with hundreds of small streaming multiprocessors, Trainium uses fewer but much larger and more capable NeuronCores — reducing control overhead and maximizing efficiency for large-model training.

NeuronCore-v2 · Compute Engines

🔢 Tensor Engine

128×128 systolic array optimized for GEMM, convolution, and transposition. Reads from SRAM buffer (SBUF), outputs to partial-sum buffer (PSUM). Loops over K-dimension for complete matrix multiplications.

📊 Vector Engine

SIMD-parallel execution of element-wise operations across multiple inputs to multiple outputs. Handles activations, normalization, and other vector-parallel workloads with high throughput.

⚡ Scalar Engine

Accelerates scalar operations where each output depends on a single input element. Provides hardware acceleration for non-linear functions like GeLU and Sqrt — reducing software emulation overhead.

🛠 GpSimd Engine

Turing-complete general-purpose SIMD cores that execute arbitrary C++ code. Enables custom operators (e.g., triangular masking for attention) without CPU round-trips. Supports future operator extensibility.

Dedicated Collective Communication

NeuronCore-v2 includes dedicated hardware communication cores separate from compute resources. This enables compute-communication overlap — gradient all-reduce and other collective operations proceed simultaneously with forward/backward passes without contention.

Large Core Architecture

Trainium uses 2 large NeuronCores per chip versus GPU architectures with hundreds of small tensor cores (e.g., 528 on H100 SXM). Larger cores reduce control plane overhead and provide better utilization for GenAI workloads characterized by large matrix multiplications.

The smallest instance hosts a single Trainium chip with 1 NeuronCore-v2. Ideal for model experimentation, debugging, small model training, and pipeline development before scaling to larger configurations. NeuronLink is not available at this size.

trn1.32xlarge — Full Node

16 Trainium chips connected via NeuronLink-v2 in a 2D Torus topology. All 512 GB of HBM is directly addressable from any chip, enabling efficient tensor parallelism and pipeline parallelism for training 100B+ parameter models on a single node.

Local Storage & EBS: Trn1 instances support up to 8 TB of local NVMe SSD storage for fast dataset access, and up to 80 Gbps of Amazon EBS bandwidth. This avoids storage bottlenecks when training on large datasets that need to be shuffled and streamed efficiently during the training loop.

Numerical Formats

Mixed Precision & Data Types

Trainium supports a rich set of numeric formats carefully chosen to balance compute throughput, memory efficiency, and training accuracy — including the novel configurable FP8 format unique to AWS silicon.

cFP8

Configurable FP8 — novel AWS format where the exponent/mantissa split is tunable per-layer by the compiler. Maximum throughput with controllable accuracy.

TrainingNEW

BF16

BFloat16 with the same exponent range as FP32. Optimal for training stability in transformer models. The primary training format on Trn1.

Primary

FP16

Half-precision floating point. Supported with stochastic rounding for improved accuracy versus standard round-to-nearest. Compatible with CUDA frameworks.

Inference & Training

TF32

TensorFloat-32: 10-bit mantissa, 8-bit exponent. Provides near-FP32 accuracy with FP16-level throughput on the Tensor Engine systolic array.

Mixed precision

FP32

Full single-precision floating point for master weights, gradient accumulation, and optimizer states where numerical precision is critical.

Master weights

INT8 / UINT8

8-bit integer formats for inference quantization. The Tensor Engine delivers 420 INT8 TOPS per chip — more than 2× the BF16 TFLOPS throughput.

Inference

Stochastic Rounding

Hardware-accelerated stochastic rounding mode prevents systematic bias in gradient accumulation. Achieves higher accuracy than legacy round-to-nearest at low precision.

Distributed Training

Parallelism Strategies

Large model training on Trn1 uses a combination of parallelism strategies, exploiting both NeuronLink intra-node and EFAv2 inter-node communication to efficiently partition models too large to fit on a single chip or instance.

↔️

Tensor Parallelism (TP)

Splits individual tensors (e.g., weight matrices) across multiple NeuronCores. Requires high-bandwidth, low-latency communication — ideal for NeuronLink-v2. Reduces per-chip memory footprint proportionally to TP degree.

Intra-nodeNeuronLink
⬛

Pipeline Parallelism (PP)

Assigns consecutive model layers to different chips or nodes. Uses micro-batching to keep all stages busy. Lower communication bandwidth requirement than TP — suitable across EFAv2 inter-node connections.

Inter/intra-nodeEFAv2
📦

Data Parallelism (DP)

Replicates the model across chips, each processing different data shards. Gradients are synchronized via AllReduce. ZeRO-1 shards the optimizer state across DP ranks to reduce memory overhead.

ZeRO-1AllReduce

3D Parallelism on Trn1: Large models like GPT-style LLMs are trained with all three parallelism strategies simultaneously — Tensor Parallelism within a node over NeuronLink, Pipeline Parallelism across nodes over EFAv2, and Data Parallelism for throughput scaling. The Neuron SDK's NxD Training library handles the configuration automatically.

Pre-training and fine-tuning of GPT-style and encoder-decoder LLMs including Llama, BERT, T5, and GPT-2/3 architectures. Trn1 customers have successfully pre-trained 13B and 70B parameter models.

When to use Trn1: Trn1 remains an excellent choice for models up to ~70B parameters, fine-tuning workloads, and teams with established Neuron SDK workflows. For frontier-scale training (100B+ parameters, MoE at trillion scale) or maximum inference throughput, Trn2 or Trn3 are recommended.
