Chip Architecture

AWS's First 3nm AI Chip

Trainium3 is built on TSMC's N3P process node — the most advanced process used in any shipping AI accelerator. A dual-chiplet design integrates 8 NeuronCore-v4 units with 144 GB HBM3e and NeuronSwitch-v1 fabric, purpose-built for MoE, reasoning, and video workloads.

⬡

NeuronCore-v4

Each chip integrates 8 NeuronCore-v4 units. Every core contains a 512×128 MXFP8/FP8 Systolic Array (2× Trn2's 256×128), a 128×128 BF16 Systolic Array, Vector, Scalar, and GPSIMD engines. Total: 32 MiB SBUF per core, 256 MiB SRAM across all 8 cores per chip.

512×128 MXFP8 Array 128×128 BF16 Array GPSIMD · NKI 256 MiB SRAM/chip
▪

HBM3e at 9.6 Gbps

Four HBM3e stacks per chip at 9.6 Gbps pin speed — the highest in any shipping AI accelerator. AWS switched from Samsung (5.7 Gbps) to Hynix/Micron HBM3e, achieving a 70% bandwidth increase. 4.9 TB/s total per chip across 144 GB.

144 GB per chip 4.9 TB/s bandwidth Hynix/Micron 9.6 Gbps
⇄

NeuronSwitch-v1

The first-ever NeuronSwitch fabric replaces Trainium2's 2D torus with an all-to-all topology, delivering 2 TB/s chip-to-chip bandwidth via PCIe Gen 6 (64 Gbps/lane). Specifically engineered for Mixture-of-Experts routing where any token must reach any expert on any chip.

All-to-all topology 2 TB/s per chip PCIe Gen 6
⌬

TSMC N3P · Dual-Chiplet

Built on TSMC's N3P process — first AI accelerator on 3nm-class silicon. Dual-chiplet design: 4 NeuronCore-v4 per chiplet, each connected to 2 HBM3e stacks. 128 hardware DMA engines shared between chiplets for maximum data movement throughput.

3nm process Dual-chiplet 128 DMA engines

Compute Engines

NeuronCore-v4 — Four Engines

Four specialized engines operate in lockstep inside each NeuronCore-v4. The headline addition is a 512×128 MXFP8 systolic array — double Trn2's FP8 array — combined with fully programmable GPSIMD processors exposed via NKI.

Tensor Engine
2× FP8 vs Trn2
BF16 mode: 128×128 systolic array. MXFP8/FP8 mode: 512×128 systolic array — 2× the contraction dimension vs Trn2's 256×128, delivering double the FP8 TFLOPS. Supports 4× structured sparsity (16:4) for ~4× throughput boost on sparse models. MXFP4 enables near-4× throughput over BF16. Reads from SBUF, writes partial sums to dedicated PSUM buffer.
512×128 MXFP8 Array 128×128 BF16 Array MXFP4 native 16:4 sparsity
Vector Engine
Wide SIMD
High-throughput element-wise operations: activation functions (including accelerated exp() computation), layer normalization, reduction ops, and transcendentals. Wider datapath than NeuronCore-v3. Processes output from the Tensor Engine PSUM buffer and writes results back to SBUF.
Accel exp() Layer norm Reductions Transcendentals
Scalar Engine
Control Logic
Orchestrates execution across all four engines. Handles address computation, loop control, branching, and DMA scheduling. Controls SBUF-to-HBM data movement and synchronization between the NeuronCore's internal execution pipelines. Acts as the primary scheduling unit that maximizes engine utilization.
DMA scheduling Loop control Synchronization
GPSIMD Engine — NKI Target
Fully Programmable
General-Purpose SIMD processors with integrated DMA — fully programmable via the Neuron Kernel Interface (NKI). Can read/write to any SBUF partition or HBM address. Exposes the complete Trainium3 ISA to developers. The Neuron Kernel Library (Apache 2.0) provides ready-to-deploy optimized kernels.
Full ISA via NKI Integrated DMA Open-source kernels
SBUF upgrade: Each NeuronCore-v4 features 32 MiB SBUF (vs 28 MiB in v3), plus a dedicated PSUM buffer. Total on-chip SRAM per chip: 256 MiB. With LNC-2 (logical pairing of two physical cores), effective SBUF doubles to 64 MiB per logical core — critical for fitting large attention intermediates without HBM round-trips.

Logical NeuronCore (LNC)

Two physical NeuronCore-v4s can be fused into one logical core, combining SBUF (64 MiB) and compute. Trainium3 Day-0 supports LNC=1 and LNC=2. LNC=8 (all 8 cores as one) is planned for mid-2026.

Why All-to-All Topology?

MoE models route each token to 2–8 expert networks spread across chips. With a ring or torus topology, most tokens require multi-hop routing that serializes traffic. NeuronSwitch-v1's all-to-all fabric gives every chip direct 1-hop access to every other chip, eliminating routing latency entirely for expert dispatch.

1-hop MoE routing No congestion Expert dispatch
🔄

Trn2 2D Torus vs Trn3 NeuronSwitch

Trainium2 used a 2D torus that excelled at dense transformer all-reduce operations (linear models). Trainium3's NeuronSwitch is optimized for sparse MoE routing patterns. The switch can still perform efficient AllReduce for dense models — so Trn3 is a strict superset of Trn2's communication capabilities.

MoE + Dense AllReduce support Strict superset

# Full ISA control via NKI import neuronxcc.nki as nki import neuronxcc.nki.language as nl import neuronxcc.nki.isa as nisa @nki.jit def flash_fwd_kernel(q, k, v, out, scale): # Direct 32 MiB SBUF access q_sbuf = nl.load(q[nl.mgrid[...]]) k_sbuf = nl.load(k[nl.mgrid[...]]) # 512×128 MXFP8 systolic array scores = nisa.nc_matmul(q_sbuf, k_sbuf) # Vector engine: accelerated exp() scores = nisa.activation(scores * scale, op=nl.exp) out_v = nisa.nc_matmul(scores, nl.load(v[...])) nl.store(out[nl.mgrid[...]], out_v)
Distributed Strategies

Parallelism on Trainium3

Expert Parallelism (MoE)

First-class on Trainium3 via NeuronSwitch-v1. Each expert shard lives on a separate chip group. The router dispatches tokens to selected experts across the all-to-all fabric. Any chip can send to any expert on any chip in a single hop.

Combines with Tensor and Pipeline Parallelism. The NeuronSwitch topology was purpose-built for this access pattern — ring/torus topologies suffer multi-hop latency for MoE routing, which NeuronSwitch eliminates.

All-to-all routingNeuronSwitch nativeMoE first-classEP + TP + PP

Tensor Parallelism

Weight matrices are sharded across NeuronCores. Each core computes its partition; results are assembled with AllGather via NeuronLink-v4. In LNC-2 mode, paired cores double their SBUF to 64 MiB, reducing HBM pressure for large attention layers.

Supported via NxD Core's ColumnParallelLinear / RowParallelLinear APIs. Essential for serving very large transformer layers that exceed a single NeuronCore's SBUF capacity.

Weight shardingAllGatherLNC-2 64 MiBColumnParallel

Pipeline Parallelism

Model layers are partitioned into stages across chips or UltraServers. 1F1B scheduling keeps all stages busy with micro-batches. NeuronLink-v4's 2 TB/s ensures activation transfers don't bottleneck inter-stage communication.

NxD Training provides automatic stage partitioning. Activation recomputation cuts HBM requirements by up to 60%, enabling models well beyond a single chip's 144 GB capacity.

1F1B scheduleMicro-batchingActivation recompute-60% HBM

Context Parallelism

Sequence length is sharded across NeuronCores — enabling training and inference with 128K+ token context windows. Trainium3's 256 MiB total SRAM per chip reduces HBM pressure for long-context attention intermediates when combined with Context Parallelism.

Critical for o1/o3-style reasoning models, multi-document retrieval-augmented generation, and multimodal architectures with dense visual token sequences. Supported in NxD Core v2.
