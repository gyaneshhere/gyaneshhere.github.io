Chip Architecture

AWS's First 3nm AI Chip

Trainium3 is built on TSMC's N3P process — the most advanced node in any shipping AI accelerator. A dual-chiplet design integrates 8 NeuronCore-v4 units, 144 GB HBM3e, and NeuronLink-v4 in a single package purpose-built for MoE, reasoning, and video generation.

NeuronCore-v4 · New Architecture
8 cores

Four-Engine Compute Core

Each NeuronCore-v4 contains: a 128×128 BF16 Systolic Array and a 512×128 MXFP8/MXFP4 Systolic Array (double the FP8 array vs Trn2), a Vector Engine with accelerated exponential computation, a Scalar Engine for control, and a GPSIMD Engine for programmable custom kernels. Total SRAM: 32 MiB SBUF + PSUM per core; 256 MiB across all 8 cores per chip.

512×128 MXFP8 Array 128×128 BF16 Array Vector Engine GPSIMD · NKI Programmable 32 MiB SBUF/core 256 MiB total SRAM

Compute Engines

NeuronCore-v4 Architecture

The fourth-generation NeuronCore features the largest MXFP8 systolic array AWS has built — a 512×128 contraction dimension that delivers double the FP8 throughput vs NeuronCore-v3, alongside fully programmable GPSIMD engines exposed via NKI.

Tensor Engine
2× FP8 vs v3
BF16: 128×128 systolic array (same size as Trn2 BF16 array). MXFP8/MXFP4: 512×128 systolic array — double the FP8 contraction dimension vs Trn2's 256×128. Reads from SBUF, writes partial sums to PSUM buffer. Loops over K dimension for complete matmul. 4× sparsity (16:4) acceleration via structured sparsity hardware. MXFP4 enables near-4× throughput over BF16.
512×128 MXFP8 Array 128×128 BF16 Array MXFP4 native 16:4 sparsity
Vector Engine
Enhanced SIMD
High-throughput element-wise operations: activation functions (with accelerated exponential computation over v3), layer normalization, reduction ops, transcendentals. Wider datapath than NeuronCore-v3. Processes output from Tensor Engine PSUM buffer and writes results to SBUF for next operation.
Accel exp() Element-wise Normalization
Scalar Engine
Control Logic
Orchestrates execution across all four engines. Handles address computation, loop control, branching, and DMA scheduling. Controls SBUF-to-HBM data movement and synchronization between the NeuronCore's internal execution pipelines. Acts as the primary scheduling unit that maximizes engine utilization.
Scheduling DMA control Synchronization
GPSIMD Engine · NKI Target
Fully Programmable
General-Purpose SIMD processors with integrated DMA — fully programmable via the Neuron Kernel Interface (NKI). Each GPSIMD can read/write to any SBUF partition or HBM address on-chip. Exposes the complete Trainium3 ISA to developers. The open-source Neuron Kernel Library provides ready-to-deploy optimized kernels built on this engine.
Full ISA via NKI Integrated DMA Open-source kernels
SBUF / PSUM: Each NeuronCore-v4 has 32 MiB SBUF (scratchpad buffer) plus a dedicated PSUM (partial sum) buffer. With 8 cores per chip, total on-chip SRAM is 256 MiB — enough to hold large attention intermediates without HBM round-trips. LNC-2 mode combines two physical cores, doubling available SBUF to 64 MiB per logical core.

# NKI: Full ISA control on NeuronCore-v4 GPSIMD import neuronxcc.nki as nki import neuronxcc.nki.language as nl import neuronxcc.nki.isa as nisa @nki.jit def flash_fwd_kernel(q, k, v, out, scale): # Direct SBUF-level access — 32 MiB per NeuronCore-v4 q_sbuf = nl.load(q[nl.mgrid[...]]) k_sbuf = nl.load(k[nl.mgrid[...]]) # 512×128 MXFP8 systolic array → PSUM attn_raw = nisa.nc_matmul(q_sbuf, k_sbuf) attn_scl = nisa.tensor_scalar(attn_raw, scale) # Vector engine: exp() with hardware acceleration attn_sft = nisa.activation(attn_scl, op=nl.exp) out_sbuf = nisa.nc_matmul(attn_sft, nl.load(v[...])) nl.store(out[nl.mgrid[...]], out_sbuf)
Distributed Training Strategies

Parallelism on Trainium3

Expert Parallelism (MoE)

First-class support on Trainium3 via NeuronSwitch-v1's all-to-all fabric. Each expert shard can be routed to any chip — critical for sparse MoE models like Mixtral, Qwen3-MoE, and Llama 4 where tokens fan-out to selected experts across the fabric.

Combines with Tensor Parallelism within each expert and Pipeline Parallelism across UltraServers. The all-to-all NeuronSwitch topology was specifically designed for this routing pattern.

All-to-all routing NeuronSwitch optimized MoE native EP + TP + PP combined

Tensor Parallelism

Weight matrices are sharded across NeuronCores or chips. Each unit computes its partition and results are assembled via AllGather over NeuronLink-v4. In LNC-2 mode, pairs of cores double their effective SBUF, reducing HBM pressure for very large attention heads.

Critical for serving large transformer layers where a single chip's SBUF cannot hold all intermediate activations. Supported via NxD Core's ColumnParallelLinear API.

Weight sharding AllGather LNC-2 amplified ColumnParallel

Pipeline Parallelism

Model layers are partitioned into stages across chips or UltraServers. 1F1B scheduling with micro-batches maximizes stage utilization. Activation recomputation cuts HBM requirements by up to 60%.

NxD Training provides automatic stage partitioning with NeuronLink-v4's 2 TB/s bandwidth ensuring activation transfers don't bottleneck pipeline throughput between stages.

1F1B schedule Micro-batching Activation recompute -60% HBM

Context Parallelism

Sequence length is sharded across NeuronCores, enabling training and inference with extremely long context windows (128K+ tokens). Critical for long-context reasoning models like o1/o3-style systems and multimodal architectures with large visual token sequences.

On Trainium3, the 256 MiB total SRAM per chip dramatically reduces HBM pressure for long-context attention intermediates when combined with Context Parallelism, enabling larger effective batch sizes at long sequence lengths.

Sequence sharding 128K+ context Reasoning models 256 MiB SRAM / chip
